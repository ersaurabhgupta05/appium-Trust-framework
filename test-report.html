🐛 Bug Report

BUG-01  MEDIUM  Backup word-chips have no accessibility IDs

Steps to Reproduce
1. Create new wallet → tap "Back Up Now" → view recovery phrase → tap "Continue"
2. Observe the word-verification chip elements in Appium Inspector
Expected
Each word chip has a unique content-desc / accessibility label (e.g. "word_chip_abandon")
Actual
Chips have no accessibility ID — they can only be targeted by XPath index, making automation brittle and accessibility tools (screen readers) non-functional.
Severity Impact
Automation blocker + accessibility compliance failure (WCAG 1.3.1)

BUG-02  MEDIUM  Token toggle elements missing accessibility labels

Steps to Reproduce
1. Home → "Manage Tokens" → search "USDC"
2. Inspect toggle with Appium Inspector
Expected
Toggle has content-desc = "Add Token Toggle_USDC" (or similar per token)
Actual
Toggle is a raw Switch view with no accessibility ID; automation must fall back to position-based XPath.
Severity Impact
Automation brittle; screen reader users cannot identify which token a toggle controls.

BUG-03  HIGH  Send form accepts "0" as a valid amount

Steps to Reproduce
1. Home → Send → enter any valid address → Continue
2. Enter "0" in the Amount field → tap Continue
Expected
Validation error: "Amount must be greater than 0"
Actual
App proceeds to the fee confirmation screen with 0 ETH amount. A user could sign a zero-value transaction wasting gas.
Severity Impact
High — financial impact: gas fees spent for no value transfer; potential UX confusion.

⚖️ Framework Choices & Trade-offs

  • Appium + WebdriverIO vs Detox: Detox is faster (grey-box, direct JS bridge) but requires the app to be built with Detox instrumentation — not possible on release APKs. Appium works on any binary, making it suitable for testing the real distribution build.
  • Android-first: iOS requires a paid Apple Developer account and macOS runner. The selector strategy (accessibility IDs) was deliberately kept cross-platform so switching to XCUITest is a capability change only.
  • Mocha over Jasmine / Jest: Mocha has native async/await support and rich before/after hooks that match mobile test lifecycle needs (app reset between suites).
  • No real transactions: All send tests stop before final confirmation. This avoids unrecoverable mainnet fund loss while still exercising 100% of the validation and UI logic.
  • Allure for reports: Allure gives interactive HTML reports with step-level screenshots and timelines. The trade-off is an extra install step (allure-commandline).
  • Environment variables for seed phrases: Never hardcode real keys. Test vectors use the publicly-known BIP-39 "test" mnemonic. Real QA environments should use ephemeral testnet wallets provisioned via CI secrets.
